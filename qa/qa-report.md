Issues, severity, reproduction, remediation

1) button-name (critical)
   - Severity: Critical
   - Files/areas: Header action icons and inline icon buttons across UI (see selector .icon-btn)
   - Reproduction:
     1. Load http://localhost:3000
     2. Tab through header to icon-only button(s) or use screen reader to move to controls
     3. Observe screen reader announces "button" with no accessible name
   - Impact: Screen reader users cannot determine button purpose; critical for usability
   - Recommended fix:
     - Provide accessible names for all interactive controls. For icon-only buttons, add aria-label or visually-hidden text. Example:
       <button class="icon-btn" aria-label="Open settings">...</button>
     - Alternatively use <span class="sr-only">Settings</span> inside the button and ensure it is not display:none.
   - Notes: Do NOT remove accessible names for icons; ensure localization if app is multilingual.

2) color-contrast (serious)
   - Severity: Serious
   - Files/areas: Primary CTA (.cta), some secondary text colors
   - Reproduction:
     1. Inspect CTA text and background with devtools; compute contrast using color picker or run axe/Lighthouse
     2. Observe contrast ratio < 4.5:1 for normal text
   - Impact: Users with low vision may not read content
   - Recommended fix:
     - Increase contrast by darkening the text or lightening the background.
     - Use WCAG contrast ratio >=4.5:1 for normal text, >=3:1 for large text.
     - Tools: use https://webaim.org/resources/contrastchecker/

3) label (serious)
   - Severity: Serious
   - Files/areas: Search input missing <label>
   - Reproduction:
     1. Navigate to the search form with keyboard or screen reader
     2. Observe input announced without label
   - Recommended fix:
     - Add an associated <label for="search">Search</label> element, or add aria-label/aria-labelledby.
     - Ensure placeholder is not used as the only label.

4) bypass (moderate)
   - Severity: Moderate
   - Files/areas: No skip link present
   - Reproduction:
     1. Use keyboard to tab from the top of the page
     2. Observe several navigation links must be traversed before reaching main content
   - Recommended fix:
     - Add a "Skip to main content" link as the first focusable element that becomes visible on focus. Example:
       <a href="#main-content" class="sr-only sr-only-focusable">Skip to main content</a>
     - Ensure the #main-content anchor exists on the page wrapping primary content.

5) document-title (minor)
   - Severity: Minor
   - Files/areas: index.html missing <title>
   - Reproduction:
     1. Open page in browser tab
     2. Observe title shows file path or default value
   - Recommended fix:
     - Add a meaningful <title> element to index.html
